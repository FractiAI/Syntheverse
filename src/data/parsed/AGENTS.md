# Parsed Content Agents

## Purpose

Storage and management of parsed PDF content chunks for the RAG pipeline in the Syntheverse system.

## Key Modules

### PDF Parsing (`*.json`)

- **Text Chunks**: Parsed text content from PDF documents
- **Chunk Indexing**: Sequential chunk numbering within documents
- **Metadata Storage**: PDF source information and processing details
- **JSON Structure**: Standardized format for parsed content

### Parsing Process

- **PDF Text Extraction**: Text content extraction from PDF files
- **Chunk Generation**: Document splitting into manageable text chunks
- **Metadata Attachment**: Source file information and processing timestamps
- **Duplicate Handling**: Skip already parsed PDF files

## Integration Points

- Parsed files generated by RAG parser in `src/api/rag-api/parser/`
- Input for RAG vectorizer in `src/api/rag-api/vectorizer/`
- Used by RAG API for semantic search and retrieval
- Connected to PDF storage in `src/data/pdfs/`

## Development Guidelines

- Maintain consistent JSON schema across all parsed files
- Include comprehensive metadata for traceability
- Ensure chunk sizes are optimized for embedding models
- Handle PDF parsing errors gracefully
- Document parsing parameters and chunking strategy

## Common Patterns

- JSON file structures for parsed content
- Chunk indexing and sequencing
- Metadata attachment to content
- PDF-to-JSON file naming conventions
- Processing pipeline integration
