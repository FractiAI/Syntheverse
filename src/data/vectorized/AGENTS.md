# Vectorized Embeddings Agents

## Purpose

Storage and management of vectorized embeddings for semantic search and retrieval in the Syntheverse RAG system.

## Key Modules

### Embedding Storage (`embeddings/`)

- **Vector Files**: JSON files containing vectorized text chunks
- **Embedding Arrays**: Numerical vector representations of text
- **Chunk Metadata**: Original text, chunk index, and processing information
- **Document Mapping**: One JSON file per source PDF document

### Metadata Storage (`metadata/`)

- **Processing Statistics**: Vectorization run statistics and performance metrics
- **Model Information**: Embedding model details and configuration
- **Quality Metrics**: Embedding quality and processing success rates

### Vectorization Process

- **Local Models**: HuggingFace models for offline embedding generation
- **Batch Processing**: Efficient processing of multiple documents
- **Skip Logic**: Automatic skipping of already vectorized files

## Integration Points

- Embedding files generated by RAG vectorizer in `src/api/rag_api/vectorizer/`
- Loaded by RAG API in `src/api/rag_api/api/` for semantic search
- Input from parsed content in `src/data/parsed/`
- Referenced by RAG API for document retrieval and ranking

## Development Guidelines

- Maintain consistent embedding schema across all files
- Include comprehensive metadata for model traceability
- Optimize embedding dimensions for performance
- Ensure local processing without external API dependencies
- Document model selection and embedding parameters

## Common Patterns

- JSON embedding file structures
- Vector array storage and formatting
- Metadata attachment to embeddings
- Batch processing workflows
- Model versioning and tracking
