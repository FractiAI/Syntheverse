# Metadata Management Agents

## Purpose

Metadata storage and management for scraping operations, processing results, and data tracking in the Syntheverse system.

## Key Modules

### Scraping Metadata (`zenodo_scrape_results.json`)

- **Zenodo Repository Data**: Metadata from Zenodo repository scraping
- **Publication Information**: Research paper metadata and identifiers
- **Download Statistics**: File download information and URLs
- **Repository Structure**: Zenodo record organization and metadata

### Test Results (`test_scrape_results.json`)

- **Test Scraping Data**: Metadata from test scraping operations
- **Validation Results**: Test data for scraper validation
- **Processing Statistics**: Test run statistics and results

### Processing Metadata

- **Data Processing Results**: Metadata from PDF processing operations
- **File Information**: Processed file metadata and statistics
- **Quality Metrics**: Data quality and processing success rates

## Integration Points

- Metadata files generated by RAG scraper in `src/api/rag-api/scraper/`
- Used by RAG parser for processing tracking
- Referenced by RAG API for data statistics
- Connected to data processing pipeline in `src/data/`

## Development Guidelines

- Maintain consistent JSON structure across metadata files
- Include timestamps and version information
- Document metadata schema and field definitions
- Ensure metadata files are properly excluded from version control
- Update metadata processing when scraper changes

## Common Patterns

- JSON metadata file structures
- Timestamp and versioning information
- Data quality metrics tracking
- Processing statistics and results
- Repository and publication metadata
