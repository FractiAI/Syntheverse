# Test Results Agents

## Purpose

Storage for automated test execution results, reports, and artifacts generated by the Syntheverse testing framework.

## Result Files

### Test Reports

HTML and XML test reports generated by pytest:

- **test_report.html**: Interactive HTML test report with detailed results
- **test_report_single.html**: Single-page HTML report
- **test_report_phased.html**: Phased execution test report
- **python_tests.html/xml**: Python-specific test results

### Log Files

Execution logs from test runs:

- **all_tests.log**: Complete test suite execution log
- **python_tests.log**: Python test execution log
- **{component}_test_*.log**: Component-specific test logs

## Integration Points

### Input Sources

- **pytest Framework**: Generates HTML/XML reports
- **Test Runners**: Executes test suites and logs output
- **CI/CD Pipeline**: Runs automated tests
- **Development Workflow**: Manual test execution

### Output Consumers

- **Quality Assurance**: Reviews test results and coverage
- **CI/CD Dashboard**: Displays test status and trends
- **Development Team**: Analyzes test failures and issues
- **Documentation**: Includes test results in reports

## Usage Patterns

### Local Testing

```bash
# Run tests with result generation
cd tests && ./run_tests.sh --all --html-report

# View test results
open results/test_report.html
```

### CI/CD Integration

```yaml
# GitHub Actions example
- name: Run Tests
  run: |
    cd tests
    ./run_tests.sh --all --junit-xml
- name: Upload Results
  uses: actions/upload-artifact@v3
  with:
    name: test-results
    path: tests/results/
```

## Development Guidelines

- Maintain test result history for trend analysis
- Document test failures with clear reproduction steps
- Archive significant test runs for regression analysis
- Clean up outdated result files periodically
- Ensure result files are gitignored appropriately
